# 新能源节点类型选择说明

## 用户问题

> 新能源是不是应该设置为PQ节点？毕竟他们也不承担电压控制的任务

**回答：完全正确！** 新能源场站应该设置为 **PQ节点**（Pctrl_mode='0'）。

## 电力系统理论基础

### 节点类型定义

| 节点类型 | 已知量 | 未知量 | 适用设备 |
|---------|--------|--------|---------|
| **PQ节点** | 有功P、无功Q | 电压V、相角δ | **新能源、负荷** |
| PV节点 | 有功P、电压V | 无功Q、相角δ | 同步发电机（有AVR） |
| 平衡节点 | 电压V、相角δ | 有功P、无功Q | 大容量发电机/外部电网 |

### 为什么新能源是PQ节点？

#### 1. 物理特性决定

```
新能源发电特性：
├── 有功功率P - 由风/光资源决定，变流器按MPPT或有功指令输出
├── 无功功率Q - 由变流器控制，通常设为零或固定功率因数
└── 电压V - 由电网决定，新能源不主动调节
         ↓
    这正是 PQ节点 的定义！
```

#### 2. 实际运行方式

**调度指令形式**：
- ✅ "风电场A今天发电100MW" → 有功指令
- ❌ "风电场A把Bus10电压控制在1.02p.u." → 不会这样下达

**电压由谁控制？**
- 传统同步发电机（通过励磁调节）
- SVG/SVC 静止无功补偿器
- 并联电容器/电抗器
- 有载调压变压器

#### 3. 变流器控制策略

风电/光伏变流器的典型控制模式：

```python
# 实际风电场控制策略
control_modes = {
    'constant_pf': {  # 恒功率因数（最常见）
        'P': '由MPPT或调度指令决定',
        'Q': 'P × tan(arccos(pf))',
        'pf': '通常0.95~1.0'
    },
    'const_q': {      # 恒无功
        'P': '由MPPT或调度指令决定',
        'Q': '固定值或调度指令'
    },
    'const_v': {      # 恒电压（很少见，需要配合无功设备）
        'P': '由MPPT或调度指令决定',
        'V': '通过调节Q来控制V'
    }
}
```

注意：即使是'const_v'模式，也需要额外的无功容量支撑，单纯风机难以实现。

### CSEE模型为什么用PV节点？

查看CSEE模型发现：`Pctrl_mode='1'` (PV节点)

可能原因：
1. **模型简化** - CSEE模型可能是简化演示，不完全符合实际
2. **特殊场景** - 可能配有独立的无功补偿设备
3. **历史原因** - 早期模型设计选择

**建议**：新算例使用 PQ节点 更符合实际。

## 潮流计算结果对比

### PQ节点 vs PV节点 对结果的影响

| 对比项 | PQ节点 (Pctrl_mode='0') | PV节点 (Pctrl_mode='1') |
|--------|------------------------|------------------------|
| **并网点电压** | 自由变化，由系统决定 | 固定在给定值 |
| **无功出力** | 固定（或按功率因数） | 自动调节以维持电压 |
| **物理真实性** | ✅ 符合新能源特性 | ⚠️ 需要无功支撑设备 |
| **计算稳定性** | ✅ 更稳定 | 可能无功越限 |

### IEEE39+风电 预期结果

```
基准模型 (无新能源):
├── 总发电: ~609.7 MW
├── 总负荷: ~609.7 MW
└── Bus10电压: ~1.02 p.u.

+100MW风电 (PQ节点, pf=1.0):
├── 总发电: ~709.7 MW (+100)
├── 总负荷: ~609.7 MW (不变)
├── 网损: ~100 MW (由新能源承担)
└── Bus10电压: ~1.03 p.u. (略有升高，因为注入有功)
    ↑ PQ节点下电压由系统自然决定

+100MW风电 (PV节点, V=1.02):
├── 总发电: ~709.7 MW (+100)
├── 总负荷: ~609.7 MW (不变)
├── 网损: ~100 MW
└── Bus10电压: 1.02 p.u. (强制固定)
    ↑ PV节点下新能源需要调节无功来维持电压
```

## 建议配置

### 风电场站（推荐）

```yaml
# PQ节点配置
component: WTG_PMSG_01
parameters:
  Pctrl_mode: '0'      # PQ节点
  pf_P: 100.0          # 总有功出力 MW
  pf_Q: 0.0            # 总无功出力 Mvar (零无功)
  # 或 pf_Q: 20.0      # 如果功率因数为0.98，Q = P × tan(arccos(0.98))
  Sbase: 2.5           # 单机容量 MVA
  UnitCount: 40        # 40台 = 100MW
```

### 带无功补偿的风电场（可选）

```yaml
# 如果场站配有SVG/SVC，可以模拟PV节点效果
component: WTG_PMSG_01
parameters:
  Pctrl_mode: '1'      # PV节点（需要无功补偿设备配合）
  pf_P: 100.0          # 有功出力
  V_cmd: 1.02          # 目标电压 p.u.
  # 无功由场站SVG自动调节
```

## 总结

| 问题 | 答案 |
|------|------|
| 新能源应该用什么节点？ | **PQ节点** (Pctrl_mode='0') |
| 为什么？ | 新能源不承担电压控制，只输出有功和无功 |
| 潮流计算有影响吗？ | 有，PQ节点更稳定，结果更符合实际 |
| 已更新算例配置？ | ✅ 已改为 PQ节点 |

**最终配置**：
```python
Pctrl_mode = '0'  # PQ节点 ✓
Pctrl_mode = '1'  # PV节点（仅特殊场景）
```

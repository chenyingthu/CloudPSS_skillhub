# IEEE39 新能源接入 live 对比报告

日期: 2026-04-03

## 目的

确认当前已经修复并重新验证过的新能源接入算例，是否与原始 `model/holdme/IEEE39` 做过有效对比，以及新能源接入后到底带来了哪些实际差异。

本报告只使用 2026-04-03 当天通过真实 CloudPSS API 重新验证过的分支：

- 基准模型: `model/holdme/IEEE39`
- 风电模型: `model/holdme/test_ieee39_wind`
- 光伏模型: `model/holdme/test_ieee39_pv`
- 混合模型: `model/holdme/test_ieee39_hybrid`

## 重要边界

- 仓库里历史上确实存在多份“新能源对比分析”文档和脚本。
- 但其中相当一部分建立在清理前的坏分支上，不能直接作为当前主线证据。
- 本报告使用的是清理后的同名分支，所有数据都来自 2026-04-03 的 fresh live `runPowerFlow()`。

## 对比口径

由于新能源接入后，平衡机组会自动回调，所以“系统总发电增量”不会简单等于新能源额定容量。

因此本报告采用以下更可靠的口径：

1. 接入点新增有功出力
2. 关键母线电压变化
3. 平衡机组出力变化

关键母线：

- `bus10` -> `canvas_0_45`
- `bus14` -> `canvas_0_211`
- `bus20` -> `canvas_0_193`
- 平衡机组母线 `bus39` -> `canvas_0_35`

## 解释

### 1. 是否已经做过“与原始 IEEE39 的对比”

答案是：

- 历史上做过，但旧材料不能直接作为当前证据。
- 对当前修复后的算例，2026-04-03 已重新做过 fresh live 对比。

### 2. 新能源接入后是否真的带来了差异

答案是肯定的。

可直接确认的最小证据是：

- 风电模型中 `bus10` 从 `0 MW` 变成 `80 MW`
- 光伏模型中 `bus14` 从 `0 MW` 变成 `50 MW`
- 混合模型中 `bus10/bus14` 分别变成 `100 MW / 50 MW`

这说明新能源元件已经真实进入潮流解，而不是只出现在拓扑里。

### 3. 为什么“总发电增量”没有等于新能源额定容量

因为基准 IEEE39 中的平衡机组会自动回调。

从 live 结果可以看到：

- 风电接入后，平衡机组大约回落 `77.7 MW`
- 光伏接入后，平衡机组大约回落 `49.7 MW`
- 混合接入后，平衡机组大约回落 `146.4 MW`

这符合电力系统潮流求解的正常行为。

### 4. 电压变化方向

这批 live 结果里，接入点附近电压并没有抬升，反而略有下降：

- 风电场接入 `bus10` 后，`bus10` 和 `bus14` 电压都略降
- 光伏接入 `bus14` 后，`bus14` 也略降
- 混合接入时，`bus10/bus14` 的降幅更大

这说明在当前 IEEE39 参数与接入方式下，新能源注入带来的潮流重分布和机组回调效应，净结果表现为局部母线轻微降压。

因此，旧文档里“接入后电压应上升”的表述不能继续直接沿用，必须以本报告这轮 live 结果为准。

## 当前可采信的结论

1. 当前修复后的 `test_ieee39_wind`、`test_ieee39_pv`、`test_ieee39_hybrid` 已经完成与基准 IEEE39 的真实 live 对比。
2. 新能源接入确实改变了基准模型的潮流运行点。
3. 差异不只是“新增一个元件”，而是：
   - 接入点出现非零发电
   - 平衡机组出力回落
   - 关键母线电压出现可测变化
4. 旧历史报告中的部分定量和定性结论需要视为过期。

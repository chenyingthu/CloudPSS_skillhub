## Remaining External/Design Items

- Public CloudPSS and `https://internal.cloudpss.com` are not tested in this campaign by explicit user scope.
- `test_integration_pandapower_contract.py` is explicitly a pandapower interface-invariant suite, not a multi-server numerical comparison.
- No weak `smoke` / `needs_improvement` pytest markers or weak `Smoke test` labels remain in `cloudpss_skills_v2/tests`; the quality gate prevents reintroduction.
- The quality gate now also blocks the hidden synthetic physical-data patterns removed during the trusted-analysis review.
- Trusted golden cases now must carry source/derivation metadata. The metadata is a provenance gate, not a claim that paid standards text has been exhaustively reproduced.
- Golden config artifacts now exist as reusable skill inputs, but all current golden configs explicitly declare `engine_runnable: false`; none are claimed as CloudPSS or pandapower benchmark network cases.
- `pandapower_two_bus_radial` is the first true engine-runnable golden benchmark; it is local pandapower only and does not make any CloudPSS live-server claim.
- `cloudpss_ieee39_powerflow` is the first CloudPSS live engine and `power_flow` skill golden benchmark; it is gated and targets only `http://166.111.60.76:50001`.
- CloudPSS short-circuit on IEEE39 is not yet a trusted golden case because current returned zero/empty current data.
- `pandapower_parallel_lines_n1` is the first trusted engine-runnable security/contingency golden benchmark; it verifies base-case line sharing and N-1 thermal overload detection through the registered `contingency_analysis` skill.
- `pandapower_two_bus_radial` now also verifies `voltage_stability` skill behavior: load scaling must alter pandapower load parameters and produce the expected monotonic PV curve.
- `model_validator` now performs real local structure/topology/parameter validation and fails if powerflow/EMT phases are requested without explicit study results; it is not a substitute for live CloudPSS numerical validation.
- First local golden cases are deterministic, hand-calculable, and source/derivation-backed; they are a foundation for later large IEEE/IEC benchmark studies, not a substitute for them.
- In ordinary CI/dev environments without the private local CloudPSS token/server route, only the two live-only EMT registry-matrix cases should skip; the rest of the matrix remains mandatory.

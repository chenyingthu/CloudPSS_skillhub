# CloudPSS User Stories

本目录已从“另一项目的技能故事库”清理为一套面向当前仓库的参考材料。

## 当前定位

这里的用户故事只用于两件事：

1. 帮助当前仓库识别值得继续增强的主线能力
2. 防止遗产需求把项目带离当前 CloudPSS 主线边界

本目录不是一个独立产品的功能清单，也不再把另一套 `skills/*` 命名空间当成当前仓库的真实接口。

## 证据规则

任何故事要进入当前主线，都必须同时满足：

1. 能映射到当前仓库的真实 CloudPSS SDK / 示例脚本工作流
2. 能写出明确的结果判据，而不是只写“功能存在”
3. 若结论依赖云端行为，必须有 live 集成测试、真实探针或已记录的 live 验证支撑

以下内容不应直接视为当前仓库能力：

- 遗产文档中出现的 `n1Analysis.*`、`batchEnhanced.*`、`report.*` 等虚构或外部项目接口名
- 没有当前仓库证据支撑的“已实现”“稳定”“禁用”状态
- 超出当前范围的 IES、实时仿真、硬件联调、市场、碳分析、微电网等方向

## 当前结论

从遗产 stories 中，真正值得保留到当前项目路线里的内容主要集中在：

- 研究分支与工作副本管理
- 多场景潮流研究与对比
- 结果导出与 Markdown 研究报告
- 基于真实模型和固定判据的安全筛查
- EMT 波形研究与量测可观测性整理

真正不值得继续沿用的部分主要是：

- 另一套 skills 产品接口层表述
- 没有 live 证据支撑的“稳定能力”声明
- 与当前普通云端 EMT 主线无关的泛化平台愿景

## 使用方式

如果后续要评估一个新增工作值不值得做，先看：

1. 它能否强化 `建模 -> 潮流 -> EMT -> 结果整理` 主线
2. 它是否只在当前已验证模型和边界内宣称成立
3. 它是否会新增真实云端证据，而不只是包装一层新文案

只有三项都能答“是”，才值得优先进入当前 backlog。
